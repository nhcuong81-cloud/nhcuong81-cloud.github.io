Privacy Policy – PMC220 / DDS3366D Energy Monitor

Last updated: 2025

Ứng dụng PMC220 / DDS3366D Energy Monitor kết hợp với phần cứng gateway (ví dụ: ESP01-RS485) cho phép thu thập dữ liệu điện năng từ đồng hồ đo (PMC220, DDS3366D, v.v.) và gửi dữ liệu đó lên máy chủ (server). Ứng dụng di động sau đó kết nối tới server để truy xuất và hiển thị dữ liệu cho người dùng.

1. Tổng quan (Overview)

Chúng tôi cam kết bảo vệ quyền riêng tư người dùng. Chính sách này mô tả những loại dữ liệu nào được thu thập, mục đích sử dụng, lưu trữ, chia sẻ và cách bạn có thể liên hệ để yêu cầu xóa hoặc hỗ trợ.

2. Dữ liệu được thu thập (Data we collect)

Vì kiến trúc hệ thống gồm gateway (ESP01-RS485) → server → app, dữ liệu được thu thập chủ yếu ở phía server. Dưới đây liệt kê chi tiết:

  • Dữ liệu đo đạc điện (Meter readings): giá trị điện áp (V), dòng điện (A), công suất (W), năng lượng tích lũy (Wh/kWh), hệ số công suất (PF), tần số (Hz), timestamp, và ID thiết bị/đồng hồ (meter ID). Đây là dữ liệu kỹ thuật, không phải dữ liệu cá nhân.
  • Metadata kỹ thuật: địa chỉ IP của gateway khi gửi lên (logging server-side), trạng thái kết nối, lỗi truyền tin (nhằm mục đích chẩn đoán).
  • Thông tin tài khoản (nếu bạn sử dụng hệ thống có đăng nhập): email, tên hiển thị, user ID (chỉ khi app/server hỗ trợ tài khoản người dùng). Nếu ứng dụng của bạn không yêu cầu đăng nhập thì phần này không áp dụng.
  • Logs hoạt động (optional): các bản ghi (logs) để phục vụ debug, phân tích sự cố và vận hành; logs thường được giữ ở mức tối thiểu và có thể ẩn danh hóa hoặc xóa định kỳ.
Lưu ý: nếu bạn không sử dụng tính năng đăng nhập tài khoản trên server, thì server chỉ lưu dữ liệu meter readings và metadata kỹ thuật; không lưu email hay thông tin cá nhân.

3. Mục đích sử dụng dữ liệu (Purpose)

  • Hiển thị dữ liệu đo đạc trên ứng dụng để người dùng giám sát điện năng theo thời gian thực hoặc lịch sử.
  • Lưu trữ lịch sử đo để báo cáo, xuất CSV, biểu đồ, phân tích tiêu thụ.
  • Giám sát hoạt động gateway/thiết bị và chẩn đoán sự cố (ví dụ: kết nối RS485, lỗi truyền dữ liệu).
  • Nếu có tài khoản: quản lý người dùng, phân quyền truy cập, và lưu cấu hình cá nhân.

4. Chia sẻ dữ liệu (Sharing & Disclosure)

  • Chúng tôi không bán dữ liệu meter readings hoặc thông tin người dùng cho bên thứ ba.
  • Server có thể chia sẻ dữ liệu với nhà cung cấp dịch vụ cloud/host (ví dụ: hosting provider) để vận hành hệ thống — trong trường hợp đó nhà cung cấp hosting chỉ xử lý dữ liệu theo yêu cầu phía chúng tôi và chịu trách nhiệm bảo mật theo hợp đồng.
  • Trong trường hợp pháp lý (ví dụ cơ quan có thẩm quyền yêu cầu), chúng tôi có thể cung cấp các bản ghi cần thiết theo quy định pháp luật.

5. Lưu giữ dữ liệu (Retention)

Chúng tôi lưu dữ liệu meter readings và logs trên server trong thời gian [thời gian lưu mặc định hoặc theo cấu hình]. Bạn có thể yêu cầu xóa dữ liệu lịch sử hoặc yêu cầu xóa tài khoản (nếu có) bằng cách liên hệ email bên dưới. Thời gian xóa có thể phụ thuộc vào quy trình sao lưu và chính sách lưu trữ của hosting.

6. Bảo mật (Security)

  • Server nên được cấu hình sử dụng HTTPS cho API giữa app và server. (Cần thiết: https://REPLACE_WITH_YOUR_SERVER_DOMAIN).
  • Dữ liệu meter readings được lưu trên server với quyền truy cập giới hạn; chỉ các dịch vụ và tài khoản được ủy quyền mới có thể truy xuất.
  • Thông tin đăng nhập (nếu có) được lưu dưới dạng băm/mã hoá theo chuẩn an toàn (ví dụ: bcrypt) — không lưu mật khẩu dạng plain-text.
  • Chúng tôi khuyến nghị bật cơ chế sao lưu, cập nhật bảo mật định kỳ và theo dõi truy cập bất thường.

7. Quyền người dùng (Your rights)

Bạn có thể:

  • Yêu cầu truy cập dữ liệu mà server lưu cho bạn (nếu có tài khoản).
  • Yêu cầu xóa dữ liệu (ví dụ: xóa lịch sử đo hoặc xóa tài khoản).
  • Yêu cầu chỉnh sửa thông tin tài khoản (nếu áp dụng).

Privacy Policy – English (Server-based architecture)

Last updated: 2025

This Privacy Policy describes how the PMC220 / DDS3366D Energy Monitor mobile application and associated server handle data. The system architecture: ESP01-RS485 reads Modbus RTU from meters → gateway sends data to server → server stores meter readings → mobile app fetches data from server.

1. Data Collected

  • Meter readings: voltage (V), current (A), power (W), energy (Wh/kWh), power factor, frequency, timestamp, meter ID.
  • Technical metadata: gateway IP (for logging), connection status, error logs.
  • User account info (optional): email, display name, user ID — only if your deployment uses user accounts.
  • Operational logs (optional): used for debugging and maintenance.

2. How We Use Data

Data is used to display measurements on the mobile app, provide historical reports, diagnostics, and for user account management if applicable. Data processing happens on the server and is accessed by the mobile app via secured API.

3. Sharing

  • We do not sell meter data or user information.
  • Hosting providers may process data under contract.
  • Data may be disclosed to authorities if legally required.

4. Retention & Deletion

Data retention periods depend on server configuration. Users may request data export or deletion via contact email.

5. Security

Use HTTPS for app−server API. Store credentials securely. Limit access to authorized services and personnel.
